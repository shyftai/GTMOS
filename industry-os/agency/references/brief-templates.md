# Brief Templates

Templates for different types of client project briefs. Load before running `/agency:brief`. Always validate a completed brief has: goal, audience, deadline, and approval contact before starting work.

---

## Campaign Brief

Use for: paid search campaigns, paid social campaigns, email campaigns, outbound sequences.

```
Campaign name: [descriptive name — client, channel, offer, quarter]
Client: [client name]
Account manager: [name]
Date: [date brief created]

Campaign goal
Primary metric: [e.g. "50 demo requests via paid search in 30 days"]
Secondary metric: [e.g. "CPL < $120"]

Target audience
Who: [persona or segment from CLIENT-BRIEF.md]
Where: [platform, geography]
List or audience source: [existing list / lookalike / interest targeting / keyword]

Offer / promotion
[What is being promoted? Is there a specific offer, incentive, or hook?]

Key messages (3 maximum)
1. [Core message — what is the most important thing to communicate?]
2. [Supporting message]
3. [Differentiator or proof point]

Call to action
Primary CTA: [e.g. "Book a free 30-minute strategy call"]
CTA URL: [verified URL — do not invent]

Channels
[e.g. Google Search, Meta, Email, LinkedIn]

Creative requirements
Formats needed: [e.g. "1080x1080, 1200x628, 9x16 Story"]
Static or video: [static / video / both]
Brand assets available: [yes — location / no — request from client]

Timeline
Brief confirmed: [date]
First draft due: [date — check SLAs in delivery-standards.md]
Client review deadline: [date]
Campaign live: [date]

Budget
Ad spend budget: $[X] (managed by client / managed by agency with markup)
Agency fee for this campaign: [included in retainer / $X one-time]

Approval contact
Name: [name]
Role: [title]
Deadline to respond: [date]

Constraints
[Brand constraints, legal restrictions, things to avoid, past campaigns that did not work]
```

## Content Brief

Use for: SEO articles, blog posts, thought leadership, website copy.

```
Content title (working): [title or working title]
Client: [client name]
Writer: [assigned team member]
Date: [date]

Target keyword
Primary keyword: [exact phrase — pulled from keyword research]
Secondary keywords: [2–3 supporting terms]
Monthly search volume: [X]
Keyword difficulty: [score or Low/Medium/High]

Search intent
[Informational / Navigational / Commercial / Transactional]
[What is someone trying to do when they search this? What do they want to find?]

Target audience
[Who is this for? Be specific — use persona profiles from CLIENT-BRIEF.md]

Key points to cover
1. [Point — this is a required section, not optional]
2. [Point]
3. [Point]
4. [Point — add as many as needed]

Angle / unique perspective
[What makes this piece different from what already ranks? Why would someone read this over existing content?]

Word count target
[X words — reference delivery-standards.md for typical ranges by intent type]

Structure
H1: [working title]
H2: [section heading]
H2: [section heading]
H2: [section heading]
[Outline as specifically as possible — the writer should not need to make structural decisions]

SEO requirements
- Include primary keyword in H1, first 100 words, and at least one H2
- Meta description: write one (150–160 characters, includes keyword, has CTA)
- Internal links: suggest [X] relevant pages to link to
- Images: [X] images needed — describe subjects for alt text

Examples of good content in this space
[Link to 2–3 pieces of competitor or benchmark content the writer should be aware of]

Things to avoid
[Angles that are overdone, claims we cannot make, tone that does not fit]

Approval contact
[Name and deadline]
```

## Ad Creative Brief

Use for: static ads, video ad scripts, display ads, social creative.

```
Campaign name: [parent campaign]
Brief name: [creative concept name]
Client: [client name]
Designer / creator: [assigned team member]
Date: [date]

Platform and format
Platform: [Meta / LinkedIn / Google Display / TikTok / YouTube]
Ad format: [Single image / Carousel / Story / Video / Display banner]
Sizes required: [list all dimensions]

Campaign objective
[Awareness / Consideration / Conversion]

Audience segment
[Who is seeing this ad? Pull from campaign targeting setup]

Hook (first 3 seconds / first glance)
[What should grab attention immediately? For video: the first line spoken or first visual. For static: the headline or hero element.]

Headline options (provide 3)
1. [Option — primary recommended]
2. [Option]
3. [Option]

Body copy / primary text
[Write the copy or provide a brief for the copywriter — specify length for platform]

Call to action
Button text: [Book a call / Learn more / Get started / Download / Watch now]
Destination URL: [verified URL]

Visual direction
Style: [photography / illustration / flat design / motion / UGC-style]
Subject: [what should be in the image or video — be specific enough to brief a designer]
Mood: [describe the feeling — energetic / calm / professional / playful]
Do / Do not: [specific visual instructions from brand guidelines]

Brand requirements
Logo placement: [yes — where / no]
Colors: [primary palette only / full palette / client-specified]
Font: [client brand font / system font / specify]

Compliance
[Any required disclaimers, restricted claims, or platform policy considerations]

Examples / references
[Link to 2–3 ad examples the client likes or that represent the desired direction]

Approval contact
[Name and deadline]
```

## Report Brief

Use for: weekly updates, monthly reports, QBR decks, campaign wrap-up reports.

```
Report name: [e.g. "Acme Corp — March 2026 Performance Report"]
Client: [client name]
Analyst: [assigned team member]
Date: [date report due]

Reporting period
[e.g. March 1–31, 2026]

Audience
[Client day-to-day contact / executive sponsor / both]
[What is their level of marketing sophistication? What do they care about most?]

KPIs to include
[List every metric to be reported — pull from CLIENT-BRIEF.md success metrics]

Required sections
1. Executive summary (3–5 bullet points: what happened, key wins, what needs attention)
2. [Section per channel or service line]
3. Month-on-month comparison
4. Progress vs. goals (from CLIENT-BRIEF.md)
5. Recommendations for next month
6. Appendix (raw data tables if needed)

Key narrative
[What is the main story this report should tell? What happened this month that the client needs to understand?]
[If performance was strong: what drove it and can we scale it?]
[If performance was weak: what is the honest explanation and what are we doing about it?]

Format
[Google Slides / Google Docs / Email / Looker Studio dashboard]
[Length: target [X] slides or [X] pages]

Data sources
[List every platform to pull data from: Google Ads, Meta Ads Manager, GA4, CRM, email platform, etc.]
[Confirm all data access is available before starting]

Due date
[Date — check SLAs: monthly report due within 3 business days of month close]

Approval contact
[Name — who needs to sign off before this goes to the client?]
```
