## Report 3: Quarterly Business Review (RevOps QBR)

**Audience:** Exec team, board (optionally), cross-functional leaders
**Cadence:** First 2 weeks of each quarter (reviewing prior quarter)
**Length:** 8–12 slides or 4–6 pages

**Sections:**
1. Revenue Performance — ARR, MRR, NRR, GRR vs. targets
2. Forecast Accuracy — how close was the forecast to actual?
3. Pipeline Quality — conversion rates, velocity, win rate by segment
4. Win/Loss Summary — top themes, competitive wins/losses
5. Data Quality Scorecard — CRM health, enrichment coverage, reconciliation status
6. Process Wins and Improvements — what RevOps improved this quarter
7. Next Quarter Plan — forecast, pipeline coverage, key initiatives

## Report 5: Board / Investor Revenue Report

**Audience:** Board of Directors, investors
**Cadence:** Monthly or quarterly
**Tone:** Precise, defensible, audit-ready

**Non-negotiable elements:**
- All metrics defined and methodology stated
- ARR definition matches what was in prior board report (flag methodology changes)
- Stripe reconciliation confirmed before publishing
- RevOps sign-off required before finance shares with board
- Every variance vs. prior period explained in 1 line

**Standard metrics to include:**
- ARR (EOM, with prior period comparison)
- MRR waterfall (New / Expansion / Contraction / Churned / Net New)
- NRR (LTM)
- GRR (LTM)
- New customer count
- Logo churn rate
- Quarterly forecast vs. actual (with forecast accuracy %)
- Pipeline coverage for next quarter
- CAC and CAC Payback Period (quarterly)
- Key operational metrics (data quality score, enrichment coverage) — optional but shows RevOps maturity
