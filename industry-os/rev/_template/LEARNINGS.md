# Learnings

Persistent memory for this RevOps workspace. Captures what worked, what failed, and what changed — so the same mistakes aren't repeated and the same wins can be repeated.

Load every session. Check before every forecast, data strategy, process recommendation, or CS intervention.

---

## How to add a learning

Capture a learning immediately after any significant event. Do not wait until the end of the quarter.

**Trigger events:**
- Forecast missed by > 10% → what assumption broke?
- Win/loss pattern emerged → what is the actionable implication?
- Data quality incident caused a business problem → what process failed?
- CRM integration broke and caused downstream issues → what was the root cause?
- Enrichment source proved unreliable → flag for waterfall adjustment
- Health score failed to predict churn → what signal was missing?
- Renewal saved from 🔴 Red → what intervention worked?
- Expansion pitch converted → what signal or timing made the difference?
- CS playbook failed → what was the wrong assumption?

**Format:**
```
## [YYYY-MM-DD] [Category]
**Context:** [what happened — 2–3 sentences]
**Learning:** [what this tells us — the insight, not just the fact]
**Apply when:** [specific trigger or workflow where this applies]
**Outcome:** Improved / Inconclusive / Still open
```

**Categories:** Data | Pipeline | Forecast | Win-Loss | Attribution | Retention | CS | Process | Integration
